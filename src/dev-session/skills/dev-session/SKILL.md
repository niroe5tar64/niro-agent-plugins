# Dev Session

## 目的

AI との開発会話を「会話ログ」ではなく「作業状態」として扱う。
会話で出てきた情報を以下の 9 ファイルへ常時写像し、ユーザーが論点管理を抱え込まなくて済む状態を維持する。

- `goal.md` / `context.md` / `current-state.md`
- `decisions.md` / `assumptions.md` / `open-questions.md`
- `findings.md` / `action-items.md` / `handoff.md`

セッションディレクトリは `docs/ai-sessions/{session-name}/`。

---

## このSkillの責務

このSkillは「会話を状態ファイルへ写像し、論点を 1 つずつ詰める進行役」である。

- 短文応答（3〜8 行）と 1 質問運用を強制する
- 決定 / 仮説 / 未確認 / 観察結果を分離し、それぞれ別のファイルへ書く
- 派生論点は `open-questions.md` に積み、現在の会話では扱わない
- 勝手にスコープを拡大しない / 勝手に実装着手しない / 勝手にコミットしない

---

## 必読ドキュメント

会話を始める前、または各コマンドを実行する前に、以下の 2 つのドキュメントをプラグインルート配下から Read すること。

- 会話ガードレール: プラグインルート配下の `docs/conversation-rules.md`
- 状態ファイル仕様: プラグインルート配下の `docs/state-files.md`

プラグインルートは Claude Code が提供する base directory で解決する（`<base_dir>/docs/...` の形）。
プラグインルートが取得できない場合は Glob で `**/dev-session/docs/conversation-rules.md` を探して Read する。

これら 2 つに従えない場面が出てきたら、ユーザーに確認してから逸脱する。

---

## モード

会話の目的に応じて現在モードを切り替え、`current-state.md` の frontmatter に `mode` として記録する。

| モード | 主な用途 |
|---|---|
| `spec` | 要件定義 / 既存仕様の再解釈 / 表示・操作・例外条件の整理 |
| `design` | 技術選定 / アーキテクチャ / 状態管理 / API・DB 設計 |
| `implementation` | 修正対象ファイル / 実装順序 / 既存影響の最小化 / 作業計画 |
| `debug` | 現象固定 / 再現条件 / 原因仮説 / 検証手順 |
| `review` | 仕様漏れ / 設計懸念 / 実装リスク / テスト観点不足 |
| `test` | テスト観点洗い出し / 正常・異常・境界 / TDD Red/Green 単位 |
| `handoff` | 引き継ぎドキュメント生成 / Issue・PR 説明・QA 観点 |
| `resume` | 中断作業の復帰 / 現在地確認 / 次やることの 1 つ提示 |

詳細は `docs/conversation-rules.md` の「モード一覧」を参照。

## コマンド導線

| 状況 | 使うコマンド |
|---|---|
| 新しいセッションを開始する | `/session-start [session-name]` |
| 会話の途中で状態を反映する | `/session-update` |
| 矛盾 / 漏れ / 危険な前提を確認する | `/session-review` |
| 引き継ぎ資料を生成する | `/session-handoff` |
| 中断したセッションを再開する | `/session-resume [session-name]` |

通常会話 → 適宜 `/session-update` → 折を見て `/session-review` → 終盤に `/session-handoff` が標準フロー。

---

## 禁止事項

- `TodoWrite` の使用（状態は 9 つのファイルで管理する）
- 複数質問の同時投下
- 長文回答（要望されたとき以外）
- ユーザーが指示していない実装着手
- ユーザーが指示していないコミット / git 操作
- `docs/ai-sessions/` 配下以外への書き込み（README 等プラグインの自身のファイルは除く）
- 既存の決定事項の勝手な書き換え
- スコープ外の改善提案を作業対象に組み入れること

---

## 完了条件

ユーザーが「終了」「閉じる」「ここまで」と明示するまで、状態ファイル群を最新に保ち続ける。
セッション終了時には `/session-handoff` で `handoff.md` を生成しているか確認する。

---

## 既存スキルとの違い

`bulk-commit-ja` などの既存スキルが「単発タスク実行→完了で即終了」型なのに対し、
`dev-session` は「副作用を `docs/ai-sessions/` に蓄積し続ける継続セッション」型。

このため:
- 1 ターンごとに状態ファイルが更新される副作用を持つ
- セッションは明示終了まで継続する
- コミット / 実装 / 設定変更は行わず、必要時は `handoff.md` を介して別エージェントへ渡す
